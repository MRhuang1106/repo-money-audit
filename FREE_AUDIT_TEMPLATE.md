# Free Repository Audit Template

Use this template for short public reviews. Keep it helpful, specific, and light.

## Repository

Link:

## Quick Summary

One sentence about what the project does well.

## Top Fixes

1. README positioning:
   Explain who the project is for and what problem it solves in the first 3-5 lines.

2. Quick start:
   Add exact install and run commands.

3. Usage example:
   Show one realistic input and expected output.

4. Trust signals:
   Add license, issue templates, contribution notes, security policy, and funding link where relevant.

5. Demo signal:
   Add a screenshot, terminal recording, hosted demo, or example output.

## Suggested README Order

```markdown
# Project Name

One-line value proposition.

## Why

Who this helps and what problem it solves.

## Quick Start

Install and run commands.

## Usage

One realistic example.

## Configuration

Environment variables, config files, or API keys.

## Examples

Screenshots, terminal output, or demo links.

## Contributing

How to report issues and submit improvements.

## Support

Funding, sponsorship, paid support, or contact link.
```
